# AI Prompt Testing

This file shows how to test the current restaurant-search prompt before it goes through the frontend.

The current AI stack lives in:
- `src/ai/gemini/prompt.ts`
- `src/ai/gemini/client.ts`
- `src/ai/gemini/parser.ts`
- `src/ai/gemini/safeguards.ts`
- `src/ai/groq/client.ts`
- `src/ai/groq/parser.ts`
- `src/ai/service.ts`

## What To Test First

Test in this order:

1. Generate the exact prompt and REST payload locally.
2. Paste that payload into Postman and call Gemini directly.
3. Run the local Gemini test script and inspect:
   - sanitized filters
   - raw Gemini text
   - parsed JSON payload

This helps you isolate whether the issue is:
- prompt quality
- Gemini response shape
- parser strictness
- frontend/backend wiring

## Prerequisites

Make sure `API/.env` has the key for the provider you want to test.

Current selectable providers:
- `gemini-flash` -> `gemini-2.5-flash`
- `gemini-pro` -> `gemini-2.5-pro`
- `groq-llama-3.3-70b` -> `llama-3.3-70b-versatile`

Use `--search-location` in npm commands. `--location` can be intercepted by npm itself.

## Option 1: Print The Exact Prompt

From the `API` folder:

```powershell
npm run test:gemini -- --query "ramen" --search-location "Dubai, UAE" --print-prompt
```

This prints the exact prompt generated by `buildRestaurantSearchPrompt(...)`.

To target another provider directly:

```powershell
npm run test:gemini -- --query "ramen" --search-location "Dubai, UAE" --provider gemini-pro --print-prompt
```

```powershell
npm run test:gemini -- --query "ramen" --search-location "Dubai, UAE" --provider groq-llama-3.3-70b --print-prompt
```

## Option 2: Print The Exact Postman Request Body

From the `API` folder:

```powershell
npm run test:gemini -- --query "ramen" --search-location "Dubai, UAE" --print-request
```

This prints:
- the Gemini REST endpoint URL
- the exact request body you can paste into Postman

Important:
- the backend SDK uses `config`
- the Gemini REST API uses `generationConfig`
- do not add `responseMimeType: "application/json"` while `googleSearch` tool use is enabled

## Postman Direct Test

For Gemini, create a new `POST` request in Postman:

```text
https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent?key={{GEMINI_API_KEY}}
```

Headers:

```text
Content-Type: application/json
```

Body shape:

```json
{
  "contents": [
    {
      "parts": [
        {
          "text": "PASTE_THE_PROMPT_HERE"
        }
      ]
    }
  ],
  "tools": [
    {
      "googleSearch": {}
    }
  ],
  "generationConfig": {
    "temperature": 0.2,
    "topP": 0.8,
    "maxOutputTokens": 4096
  }
}
```

What to inspect in the response:
- `candidates[0].content.parts[0].text`
- `candidates[0].groundingMetadata.webSearchQueries`
- `candidates[0].groundingMetadata.groundingChunks`

The backend parser currently accepts:
- a top-level JSON array
- or an object shaped like `{ "results": [...] }`

For Groq, create a new `POST` request in Postman:

```text
https://api.groq.com/openai/v1/chat/completions
```

Headers:

```text
Authorization: Bearer {{GROQ_API_KEY}}
Content-Type: application/json
```

Body shape:

```json
{
  "model": "llama-3.3-70b-versatile",
  "messages": [
    {
      "role": "user",
      "content": "PASTE_THE_PROMPT_HERE"
    }
  ],
  "temperature": 0.2,
  "top_p": 0.8,
  "max_completion_tokens": 4096
}
```

## Local Script Test

Use the local script when you want to test the same prompt, safeguards, Gemini client config, and parser that the backend uses.

Basic example:

```powershell
npm run test:gemini -- --query "ramen" --search-location "Dubai, UAE"
```

Gemini Pro example:

```powershell
npm run test:gemini -- --query "ramen" --search-location "Dubai, UAE" --provider gemini-pro
```

Groq example:

```powershell
npm run test:gemini -- --query "ramen" --search-location "Dubai, UAE" --provider groq-llama-3.3-70b
```

Distance-based example:

```powershell
npm run test:gemini -- --query "seafood" --search-location "Dubai Marina, UAE" --lat 25.08 --lng 55.14 --radiusKm 5 --openNow true --sort distance
```

What the script prints:
- sanitized filters
- estimated prompt token count
- raw Gemini response text
- parsed Gemini payload

## Recommended Debug Flow

1. Run `--print-request`.
2. Paste that request into Postman and send it.
3. If Gemini output is wrong, fix the prompt first.
4. If Gemini output looks usable but backend fails, run the local script without `--print-request`.
5. Compare:
   - raw Gemini text
   - parsed Gemini payload
